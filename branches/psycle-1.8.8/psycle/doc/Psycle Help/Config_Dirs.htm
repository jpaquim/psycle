Directories

(Configuration > Settings... >Directories)

Song directory: Default directory for opening/saving of .psy files.
Instrument directory: Default directory for samples (.wav and .iff).
Plug-In directory: Directory that contains your native plugins. This directory is different between the x86 and x64 releases
VST Plug-In directory: Directory that contains your VST plugins. If you have a bit bridging application, you can specify both, the 32bit Vst plugin directory and the 64bit one. Else you will be able to set only the directory of the same bits than the Psycle executable.
Right now, Psycle supports jBridge natively. Other bit bridges might work without integration.
Skin directory: Directory containing your Psycle skins.
User presets directory: Directory containing the presets for the machines. Originally Psycle saved the presets next to the dll files, but this is a bad practice in Windows Vista and 7.

A word of advice: keep your native plugin directory separate from your VST directory - you can run into problems since both types of plugin are .dll files.
